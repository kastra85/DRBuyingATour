# План автоматизации в рамках дипломного проекта по профессии Тестировщик

## Дипломный проект 

Автоматизация тестирования комплексного сервиса, взаимодействующего с СУБД и API Банка.

## Перечень автоматизированных сценариев:

1 Отправка формы для данных карты с валидными значениями в полях при оплате по дебетовой карте

Шаги:
* Открыть страницу сайта http://localhost:8080
* Нажать кнопку "Купить"
* Заполнить все поля формы для данных карты валидными значениями
* Нажать на кнопку "Продолжить"
* Проверить наличие сообщения об успешном платеже
* Проверить наличие данных о платеже в БД

Тестовые данные:
* Номер карты: 4444 4444 4444 4441 (из банковских сервисов, действующая карта)
* Месяц: 12 (срок действия действующей карты)
* Год: 24 (срок действия действующей карты)
* Владелец: IVAN IVANOV (имя и фамилия на латинице, как на банковской карте)
* Cvc/cvv: 999 (трехзначное число, как на банковоской карте)
* Информация об успешном платеже в БД:APPROVED (из банковскиз сервисов)

Ожидаемый результат:
* Страница сайта открывается
* Форма для данных карты открывается
* Поля формы для данных карты заполняется
* Заполненная форма для данных карты успешно отправляется на сервер
* После отправки формы появляется сообщение "Успешно! Платеж проведен"
* После отправки формы и появления сообщение "Успешно! Платеж проведен" в БД сохраняется информация об успешном платеже

## Перечень возможных рисков.

Возможные риски при автоматизации сценариев:
* Отсутствует подтверждение об успешном платеже при отправке формы для данных карты с валидными значениями в полях
* Отсутствует предупреждение об ошибке и о неуспешном платеже при отправке формы для данных карты с невалидными значениями в полях
* Отсутствует предупреждение об ошибке при отправке формы для данных карты без значений в полях
* Форма для данных карты с пустыми полями или с невалидными значениями отправляется на сервер
* Данные о платежах не сохраняются в СУБД
